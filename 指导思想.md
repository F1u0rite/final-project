# **项目名称：基于离线语义先验的视觉语言探索导航系统**
**(Offline Semantic-Prior Guided Vision-Language Frontier Navigation)**

## **核心理念**
*   **VLMaps (离线)**：负责 **“记忆”**。预先跑遍场景，将视觉信息压缩为一张静态的**全局语义热力图**。
*   **VLFM (在线)**：负责 **“探索”**。加载热力图，将其作为 **“引力场”**，在每一步选择边界点时，优先选择那些**既能看到新视野，又指向高概率目标区域**的方向。
*   **无全局规划**：机器人没有预设路线，它是被语义引力**吸引**着走向目标的，保留了应对动态环境和误检的灵活性。

---

## **阶段一：离线全局语义建图 (The Map Builder)**
**目标**：生成一张只读的、包含多类别语义信息的静态地图文件。此阶段不涉及具体任务指令。

### **1.1 全覆盖数据采集 (Coverage Data Collection)**
*   **输入**：场景 ID，预定义类别列表 (Vocabulary, e.g., `["chair", "bed", "table"]`)。
*   **策略**：
    *   控制机器人在场景中执行**弓字形 (Boustrophedon)** 或 **随机游走 (Random Walk)** 策略。
    *   **终止条件**：当占据网格覆盖率 > 90% 或 步数达到上限。
*   **输出**：原始数据集 `{RGB, Depth, GT_Pose}` 序列。

### **1.2 语义特征投影与融合 (Semantic Projection & Fusion)**
*   **处理逻辑**：
    1.  **CLIP 编码**：对每一帧 RGB 图像提取稠密特征，并与 Vocabulary 中**所有类别**计算相似度，生成多通道热力图。
    2.  **3D 投影**：利用 Depth 和 GT Pose 将像素级热力图反投影到全局 2D 网格。
    3.  **最大值池化 (Max-Pooling)**：
        *   对于地图上的每个网格 $(u, v)$ 和每个类别 $c$，保留历史观测中的最大置信度。
        *   公式： $\text{Map}[u, v, c] = \max(\text{Map}[u, v, c], \text{Score}_{new})$
*   **产出物**：
    *   `semantic_map.npy`: 形状 `[H, W, Num_Classes]` 的浮点数数组。
    *   `occupancy_map.npy`: 形状 `[H, W]` 的二值数组 (0=未知/自由，1=障碍)。
    *   `metadata.json`: 分辨率、原点偏移、类别顺序。

---

## **阶段二：在线语义引力导航 (The Semantic-Guided Explorer)**
**目标**：接收自然语言指令，加载离线地图，通过**改进的边界点评分机制**实现高效探索。

### **2.1 初始化与指令解析 (Init & Parsing)**
*   **输入**：
    *   用户指令 (e.g., "Go to the chair")。
    *   离线地图文件 (`semantic_map.npy`, `occupancy_map.npy`)。
*   **操作**：
    1.  **类别匹配**：将指令中的目标物体映射到离线地图的类别索引 $C_{target}$。
    2.  **提取先验场**：从 `semantic_map` 中提取第 $C_{target}$ 通道，作为**全局语义引力场 (Global Semantic Gravity Field)**。
    3.  **坐标对齐**：建立机器人实时位姿与地图网格的转换关系。

### **2.2 混合价值函数设计 (Hybrid Value Function) —— 核心创新**
这是本项目的灵魂。**不使用 A***，而是重新定义**如何评价一个边界点 (Frontier)**。

*   **候选集生成**：
    *   基于实时 Depth 和 **离线 `occupancy_map`** (用于剔除已知墙壁后的无效边界)，生成当前可行的边界点集合 $F = \{f_1, f_2, ..., f_k\}$。
*   **评分公式**：
    $$Score(f_i) = \alpha \cdot S_{local}(f_i) + \beta \cdot S_{global}(f_i) - \gamma S_{cost}(f_i)$$  
    
    *   **$S_{local}(f_i)$ (局部语义分 - 原有 VLFM 逻辑)**：
        *   模拟移动到 $f_i$ 后的视角，计算与指令的 CLIP 相似度。
        *   *作用*：确保如果目标就在眼前，能立刻被发现。
        
    *   **$S_{global}(f_i)$ (全局引力分 - 新增 VLMaps 逻辑)**：
        *   **逻辑**：评估 $f_i$ 是否位于或指向“高概率区域”。
        *   **计算方法 (推荐)**：
            1.  将 $f_i$ 的世界坐标映射到地图网格 $(u, v)$。
            2.  直接读取引力场在该位置的值: $P = \text{Field}[u, v]$  。
            3.  或者，计算从当前位置到 $f_i$ 的向量方向上，引力场的**平均势能**。
        *   *作用*：如果 $f_i$ 位于地图上“椅子”得分很高的区域，该项得分极高，吸引机器人前往。
    *   **$S_{cost}(f_i)$ (当前位置和边界点的代价)**
        *   **逻辑**：因为全局引力分可能会使更远处的边界点得分更高。
        *   *作用*：平衡局部和全局的作用。   
        
    *   **$\alpha, \beta$ 权重**：
        *   建议初始值: $\alpha=0.4,\quad \beta=0.6$  (更信赖全局先验)。
        *   **自适应机制**：如果离线地图中目标类别的最大值接近 0 (说明地图里也没见过这东西)，自动设 $\beta=0$，退化为纯 VLFM 探索。

### **2.3 探索执行循环 (Exploration Loop)**
*   **流程**：
    1.  **感知**：获取当前观测，更新局部占据网格。
    2.  **生成边界**：识别当前可探索的边界点 $F$。
    3.  **打分**：利用上述**混合价值函数**计算每个 $f_i$ 的得分。
    4.  **决策**：选择 $Score$ 最高的点 $f_{best}$。
    5.  **移动**：向 $f_{best}$ 执行直线移动 (配合简单的局部避障，如 DWA)，**不规划全局路径**。
    6.  **重规划**：每移动 $N$ 步 (或到达 $f_{best}$ 后)，**停止**，回到步骤 1 重新扫描和打分。
*   **特点**：机器人是 **“走一步看一步”**，但每一步都被离线地图的“引力”修正了方向。

## **阶段三：实验验证与对比 (Evaluation)**
**目标**：证明“有地图指引的探索”比“盲目探索”更快。

### **3.1 对照组设置**
1.  **Baseline (Pure VLFM)**:
    *   **配置**: $\beta = 0$  。完全不加载离线地图，仅靠实时视野探索。
    *   **预期**：像无头苍蝇，容易重复探索，路径曲折。
2.  **Ours (Prior-Guided VLFM)**:
    *   **配置**: $\beta > 0$  。加载离线地图，启用全局引力。
    *   **预期**：出生后即朝向目标高概率区直走，路径平滑，步数少。

### **3.2 关键指标**
*   **SPL (Success weighted by Path Length)**: 核心指标。衡量是否走得快且对。
*   **Step Count**: 到达目标所需的总步数 (预期显著减少)。
*   **Revisit Rate**: 重复访问同一网格的比例 (预期显著降低，因为全局引力避免了走回头路)。
*   **First Sight Time**: 第一次看到目标物体的时间 (体现先验知识的加速效果)。

### **3.3 可视化展示 (Visualization)**
*   **界面布局**：
    *   **左屏**：机器人第一视角 (RGB + 深度)。
    *   **右屏 (上帝视角)**：
        *   **背景层**：灰色离线占据地图。
        *   **热力层**：**红色渐变**显示目标类别的全局引力场 (来自 VLMaps)。
        *   **动态层**：
            *   **绿色点**：当前检测到的所有边界点 (Frontiers)。
            *   **绿色大号箭头**：选中的 $f_{best}$ (可以看到它总是指向红色最亮的区域)。
            *   **蓝色轨迹线**：机器人走过的路径。
*   **叙事效果**：观众能直观看到，Baseline 的绿色箭头乱指，而你的绿色箭头始终被红色热力图“吸”过去。

### **实施 Tips**
1.  **先跑通建图**：先写 `offline_builder.py`，确保能生成一张看得懂的 heatmap (比如椅子位置确实是红色的)。这是后续所有工作的基础。
2.  **调试 $\beta$ 参数**：
    *   如果 $\beta$ 太大：机器人可能死板地冲向地图中心，忽略路边的明显目标。
    *   如果 $\beta$ 太小：效果不明显，退化为随机探索。
    *   **最佳实践**：在报告中展示一组 $\alpha:\beta$ 的消融实验 (0:1, 0.5:0.5, 1:0)。
3.  **处理“空地图”情况**：代码中必须包含检查逻辑。如果指令是找一个地图里完全没记录的东西（比如地图只有 chair，指令找 apple），系统应自动检测并设 $\beta=0$，保证系统不崩溃且能退化为通用探索。

## **总结：你的核心故事线**
> “传统的 VLFM 虽然强大，但缺乏长期记忆，导致在大型场景中效率低下。我们提出了一种**解耦架构**：利用 **VLMaps** 预先构建全局语义**引力场**，并在 **VLFM** 的探索决策中注入这一先验知识。
> 
> 我们**不使用僵化的全局路径规划 (A*)**，而是通过**重塑边界点评分函数**，让机器人在保持灵活探索能力的同时，被语义引力自然地引导至目标区域。实验证明，这种方法显著减少了无效探索，提升了导航效率。”

下面是一个基于 **“离线 VLMaps 建图 + 在线 VLFM 语义引力探索”** 架构的 **全链路输入输出 (I/O) 详细清单**。

### **阶段一：离线全局语义建图 (Offline Mapping Pipeline)**
**目标**：输入原始场景数据，输出静态的语义地图文件。

#### **步骤 1.1: 全覆盖数据采集 (Coverage Data Collection)**
*   **输入 (Inputs)**:
    1.  `scene_id` (str): 场景唯一标识 (e.g., `"scene_001"`).
    2.  `coverage_strategy` (str): 覆盖策略 (e.g., `"boustrophedon"`, `"random_walk"`).
    3.  `simulator_env`: 仿真环境句柄 (用于控制机器人移动和获取传感器数据).
    4.  `max_steps` (int): 最大采集步数 (e.g., `500`).
    5.  `sensor_config` (dict): 传感器参数 (分辨率, 视场角 FOV).
*   **处理逻辑**:
    *   根据策略生成动作序列。
    *   执行动作，同步记录传感器数据。
*   **输出 (Outputs)**:
    1.  `raw_trajectory_data` (List[Dict]): 包含每一步数据的列表。
        *   每个元素结构:
            ```python
            {
                "step_id": int,
                "rgb_image": np.array (H, W, 3),      # uint8
                "depth_image": np.array (H, W),       # float32 (meters)
                "pose": np.array (4, 4),              # 4x4 齐次变换矩阵 (世界坐标系)
                "timestamp": float
            }
            ```
    2.  `trajectory_metadata.json`: 记录场景ID、总步数、采集时间。

#### **步骤 1.2: 语义特征提取与投影 (Semantic Projection)**
*   **输入 (Inputs)**:
    1.  `raw_trajectory_data`: 来自步骤 1.1 的输出。
    2.  `vocabulary` (List[str]): 预定义的语义类别列表 (e.g., `["chair", "bed", "table", "sofa"]`).
    3.  `clip_model`: 预训练的 CLIP 模型 (用于提取像素级/区域级语义特征).
    4.  `intrinsic_matrix`: 相机内参矩阵 (3x3).
*   **处理逻辑**:
    *   对每帧 RGB 进行 CLIP 编码，生成与 `vocabulary` 对应的多通道热力图。
    *   利用 Depth 和 Pose 将 2D 热力图反投影到 3D 点云，再投影到全局 2D 网格。
*   **输出 (Outputs)**:
    1.  `frame_semantic_maps` (List[np.array]): 每一帧的全局局部映射结果。
        *   形状: `(N_frames, H_map, W_map, N_classes)`
        *   数据类型: `float32` (置信度 0.0 ~ 1.0).
    2.  `frame_occupancy_masks` (List[np.array]): 每一帧的占据掩码。
        *   形状: `(N_frames, H_map, W_map)`
        *   数据类型: `bool` (True=已观测/障碍, False=未知).

#### **步骤 1.3: 地图融合与序列化 (Map Fusion & Serialization)**
*   **输入 (Inputs)**:
    1.  `frame_semantic_maps`: 来自步骤 1.2.
    2.  `frame_occupancy_masks`: 来自步骤 1.2.
    3.  `fusion_method` (str): 融合策略 (固定为 `"max_pooling"`).
*   **处理逻辑**:
    *   **语义融合**: 对 `frame_semantic_maps` 沿时间轴做 `np.max` 操作。
    *   **占据融合**: 对 `frame_occupancy_masks` 沿时间轴做 `np.logical_or` 操作。
    *   **保存**: 将结果写入磁盘。
*   **输出 (Outputs) [最终产物]**:
    1.  `semantic_map.npy`:
        *   形状: `(H_map, W_map, N_classes)`
        *   含义: 每个网格、每个类别的最大历史置信度。
    2.  `occupancy_map.npy`:
        *   形状: `(H_map, W_map)`
        *   含义: `1`=障碍/已观测墙壁, `0`=未知/自由空间 (注意：这里通常只标记绝对障碍，自由空间需结合推理).
    3.  `map_metadata.json`:
        ```json
        {
          "scene_id": "scene_001",
          "resolution": 0.05,  // 米/像素
          "origin": [x_min, y_min], // 地图左下角的世界坐标
          "shape": [200, 200],
          "classes": ["chair", "bed", "table", "sofa"],
          "creation_time": "2026-03-17T19:00:00"
        }
        ```

### **阶段二：在线语义引力导航 (Online Navigation Pipeline)**
**目标**：输入指令和离线地图，通过循环决策输出动作序列。

#### **步骤 2.1: 系统初始化与指令解析 (Initialization)**
*   **输入 (Inputs)**:
    1.  `instruction` (str): 自然语言指令 (e.g., `"Find the red chair"`).
    2.  `semantic_map.npy`: 来自阶段一的输出.
    3.  `occupancy_map.npy`: 来自阶段一的输出.
    4.  `map_metadata.json`: 来自阶段一的输出.
    5.  `start_pose`: 机器人初始位姿 (4x4 矩阵).
    6.  `clip_text_encoder`: 用于编码指令的 CLIP 文本端.
*   **处理逻辑**:
    *   加载 `.npy` 和 `.json`。
    *   将 `instruction` 编码为向量，并与 `metadata.classes` 匹配，找到 `target_class_idx`。
    *   从 `semantic_map` 中提取 `target_class_idx` 对应的通道，生成 `global_gravity_field` (2D 数组)。
    *   检查 `global_gravity_field` 的最大值。若接近 0，设置 `beta = 0`。
*   **输出 (Outputs) [状态变量]**:
    1.  `nav_state` (Dict):
        ```python
        {
          "target_class_idx": int,
          "target_class_name": str,
          "gravity_field": np.array (H, W), // 2D 引力场
          "static_occupancy": np.array (H, W), // 静态障碍图
          "alpha": 0.4,
          "beta": 0.6, // 动态调整
          "current_pose": np.array (4, 4),
          "history_trajectory": []
        }
        ```

#### **步骤 2.2: 边界点检测 (Frontier Detection) - 每 N 步执行**
*   **输入 (Inputs)**:
    1.  `current_depth`: 实时深度图 (H, W).
    2.  `current_pose`: 机器人当前位姿.
    3.  `static_occupancy`: 来自 `nav_state` 的离线障碍图.
    4.  `local_occupancy`: 实时构建的局部占据网格 (可选，用于动态避障).
*   **处理逻辑**:
    *   将 `current_depth` 投影到全局坐标系，更新局部自由空间。
    *   结合 `static_occupancy` (已知墙) 和 `local_occupancy` (新障碍)。
    *   寻找 **自由空间** 与 **未知空间** 的边界像素簇。
    *   计算每个簇的质心作为候选边界点。
*   **输出 (Outputs)**:
    1.  `frontiers` (List[Dict]): 候选边界点列表。
        ```python
        [
          {
            "id": 0,
            "world_coord": [x, y],
            "map_coord": [u, v],
            "yaw_angle": float, // 建议朝向
            "distance_from_robot": float
          },
          ...
        ]
        ```

#### **步骤 2.3: 混合价值评分 (Hybrid Scoring) - 核心步骤**
*   **输入 (Inputs)**:
    1.  `frontiers`: 来自步骤 2.2.
    2.  `instruction_embedding`: 指令的 CLIP 向量.
    3.  `gravity_field`: 来自 `nav_state` 的全局引力场.
    4.  `nav_state`: 包含 alpha, beta 等参数.
    5.  `simulator_env`: 用于模拟视角 (Render future view).
*   **处理逻辑**:
    *   遍历每个 `frontier`:
        *   **计算 $S_{local}$**: 模拟移动到该点 -> 渲染 RGB -> CLIP 编码 -> 与 `instruction_embedding` 计算余弦相似度。
        *   **计算 $S_{global}$**: 读取 `gravity_field[frontier.map_coord]` 的值 (或周围邻域的平均值)。
        *   **计算 $S_{cost}$**: 计算当前点和 `frontier` 的欧拉距离，然后进行归一化。
        *   **计算总分**: $Score = \alpha \cdot S_{local} + \beta \cdot S_{global} + \gamma \cdot S_{cost}$.
*   **输出 (Outputs)**:
    1.  `scored_frontiers` (List[Dict]): 带有分数的边界点列表。
        ```python
        [
          {
            "id": 0,
            "world_coord": [x, y],
            "score_local": 0.2,
            "score_global": 0.85,
            "score_cost": 0.3,
            "total_score": 0.44, // 假设 alpha=0.4, beta=0.6, gamma=0.5
            ...
          },
          ...
        ]
        ```

#### **步骤 2.4: 决策与执行 (Decision & Execution)**
*   **输入 (Inputs)**:
    1.  `scored_frontiers`: 来自步骤 2.3.
    2.  `current_pose`: 当前位姿.
*   **处理逻辑**:
    *   选择 `total_score` 最高的 frontier 作为 `goal_point`.
    *   生成简单的运动指令：向 `goal_point` 直线移动 (或使用局部规划器如 DWA 避开实时障碍)。
    *   执行移动 `N` 步 (或直到到达该点附近)。
*   **输出 (Outputs)**:
    1.  `action_sequence` (List[str/float]): 发送给机器人的底层控制指令 (e.g., `[move_forward, 1.0m]`, `[rotate, 30deg]`).
    2.  `updated_pose`: 执行后的新位姿.
    3.  `observation`: 新位置看到的 RGB-D 数据 (用于下一轮循环).

#### **步骤 2.5: 终止判定 (Termination Check)**
*   **输入 (Inputs)**:
    1.  `current_observation`: 最新视野.
    2.  `instruction_embedding`.
    3.  `max_steps_reached`: 是否超过总步数限制.
*   **处理逻辑**:
    *   计算当前视野与指令的相似度。若 > 阈值 (e.g., 0.8)，判定为 **Success**。
    *   若步数耗尽且未找到，判定为 **Failure**。
*   **输出 (Outputs)**:
    1.  `episode_result` (Dict):
        ```python
        {
          "success": bool,
          "total_steps": int,
          "trajectory": List[poses],
          "final_position": [x, y],
          "spl_score": float
        }
        ```

### **总结：数据流全景图**

| 步骤 | 模块名称 | 关键输入 (Key Inputs) | 关键输出 (Key Outputs) | 备注 |
| :--- | :--- | :--- | :--- | :--- |
| **1.1** | 数据采集 | 场景 ID, 策略 | `raw_trajectory_data` (RGB-D-Pose) | 离线运行一次 |
| **1.2** | 语义投影 | 轨迹数据, Vocabulary, CLIP | `frame_semantic_maps` | 耗时主要在此 |
| **1.3** | 地图融合 | 帧级地图 | **`semantic_map.npy`**, **`occupancy_map.npy`**, `metadata.json` | **阶段一最终产物** |
| **2.1** | 初始化 | 指令, `.npy` 地图文件 | `nav_state` (含 `gravity_field`) | 在线运行开始 |
| **2.2** | 边界检测 | 实时深度, 离线障碍图 | `frontiers` (候选点列表) | 每 N 步执行 |
| **2.3** | **混合评分** | 候选点, 指令向量, **引力场** | `scored_frontiers` (带总分) | **核心算法逻辑** |
| **2.4** | 决策执行 | 最高分候选点 | `action_sequence` (动作) | 驱动机器人 |
| **2.5** | 终止判定 | 实时视野 | `episode_result` (成功/失败) | 结束标志 |

这个 I/O 清单非常具体，你可以直接照着这个定义 Python 类的 `__init__` 参数和函数的 `return` 值。这样开发时就不会出现“数据对不上”的问题了。
